Transforming Meanings

How a machine interprets language

Just one word: "Harry". You think WTF is Harry? OK, let's give a little more context.

"Harry visits Hogwarts."

Boom! Just by adding 2 words the meaning of "Harry" changed. "Hogwarts" transformed the neutral person "Harry" into a young wizard "Harry Potter" visiting a magical school. Welcome to the magic of transformers.

If you remember the concept of embeddings, you might recall that each word is represented as a vector in a high-dimensional space. This is called the latent space. The latent space contains the semantics of the whole world. Every possible meaning is encoded in this space. This could be both words or sounds. The latent space of LLMs has 4096 dimension and above whilst the latent space of music only has 512 dimensions. This does not imply that one is superior to the other, though there are different opinions on this matter.

We'll take a look under the hood of modern transformers.

The Attention Mechanism

The Attention Mechanism is the architectural core of the Transformer model, enabling it to process sequential data (such as language or audio) by capturing contextual dependencies regardless of their distance in a sequence. Unlike traditional recurrent architectures that process tokens step-by-step, Self-Attention allows every token to look at, weigh, and aggregate information from every other token simultaneously.

The QKV Paradigm

This dynamic routing of information is achieved through the Query-Key-Value (QKV) framework, where linear projections map the input data into distinct functional roles:

  • Query ($Q$): Represents the current token actively searching for relevant contextual hooks.
  • Key ($K$): Represents the semantic profile or "address" offered by every token in the sequence.
  • Value ($V$): Contains the actual information payload or sensory content of the tokens.

Mathematical Inversion and Bottleneck

By computing the dot-product of the Queries and Keys ($Q \cdot K^T$), the model generates a raw compatibility matrix. This matrix is normalized via the Softmax function to produce a probability distribution, the Attention Scores, representing how much focus one token should allocate to another. Finally, these scores are multiplied by the Value matrix ($V$) to compute a weighted sum. This process physically transforms each token's static vector embedding within the latent space, updating its coordinates based on the gravity of its surrounding context, thus changing its meaning.

*Note for the architecture critique: While mathematically elegant, computing this $N \times N$ attention matrix creates a quadratic computational bottleneck relative to sequence length ($N$), which serves as the primary catalyst for modern efficiency research in alternative sequence models like SSMs.

For each of these matrices, there are corresponding weight matrices $W_Q$, $W_K$, and $W_V$. Those matrices are learned during training and are static during inference.

The Transformation Into Square Hell: Attention Scores ($Q \cdot K^T$)

We want to know how much attention each word should pay to every other word in the sentence regarding all dimensions at once. Which of the rows of the $K$ matrix is most relevant to answer the questions of the rows of the $Q$ matrix? Which word is most relevant to answer the questions of "Harry" regarding all dimensions at once?

To find the best-fitting vector of the $K$ matrix for each row of the $Q$ matrix, we compute the dot product as single value representation of the similarity between the two vectors. The value is relative. If it's high compared to other dot products, it means that the two vectors are similar and the word of the $K$ matrix is relevant to answer the questions of the word of the $Q$ matrix.

If we apply the dot product to all rows of the $Q$ matrix and all rows of the $K$ matrix, we have to transpose the $K$ matrix to align the dimensions for multiplication. That means we multiply the first row of the $Q$ matrix with the first column of the $K^T$ matrix, the second row of the $Q$ matrix with the second column of the $K^T$ matrix, and so on. The result is a square matrix of size $N \times N$, where each entry represents how much attention one word should pay to another word in the sentence. The dimensions are completely eliminated in this step, and we are left with a pure sentence-length table of attention scores. The columns and rows of the resulting matrix correspond to the words in the sentence, and the values indicate the strength of attention one word should give to another.

Except for the high values of the self attention in the diagonal, the highest value is 14.01 in the first row and third column. This means that "Harry" should pay the most attention to "Hogwarts".

The 4D Dot Product Verification for Harry -> Hogwarts:
(1.1 * 1.0) + (1.55 * 8.0) + (0.85 * 0.6) + (0.0 * 0.0) = 1.1 + 12.4 + 0.51 + 0 = 14.01

In the attention scores matrix, the difference between $N$ and $d$ becomes brutally apparent. We multiply the Queries $(3 \times 4)$ by the transposed Keys $K^T$ $(4 \times 3)$. The dimensional width $d=4$ cancels out, creating a pure, square sentence-length table: $N \times N = 3 \times 3$. That means that the matrix grows quadratically with the sentence length $N$. This is the reason why transformers have a quadratic complexity in terms of memory and computation, which can become a bottleneck for long sequences. When you hear about energy and water consumption of LLMs, this is the reason why. The attention scores matrix is the most expensive part of the transformer architecture. It demands a lot of memory and computation to calculate the attention scores for every pair of tokens in the input sequence, especially for long sequences. This is why researchers are exploring alternative architectures that can reduce this quadratic complexity, such as linear attention mechanisms or sparse attention patterns.

At this point, you also understand why LLMs try to limit the context window to a certain number of tokens. The longer the context, the more expensive the attention scores matrix becomes. At a certain time in conversation you get the message from your chatbot that this should be the last message in the conversation. The conversation gets too expensive as the costs grow quadratically with the chat length.

Final Residual Addition: Fusion of Identities

We add the context vector to the original embedding of "Harry" to get the final contextualized embedding.

x_Harry_final = x_Harry_static + x_Harry_context
x_Harry_final = [2.0, 1.0, 0.5, 0.0] + [0.1, 3.2, 0.1, 0.0]
x_Harry_final = [2.1, 4.2, 0.6, 0.0]
State of "Harry" 4D Vector Coordinates [Person, Magic, Royalty, Verb] Phenomenology in Space
Input Embedding [2.0, 1.0, 0.5, 0.0] A static noun floating semantically unconstrained in space.
Contextualized Result [2.1, 4.2, 0.6, 0.0] Uniqueness: The magic axis triumphs. Harry has absorbed the substance of Hogwarts but structurally remains a noun (verb axis stays strictly at 0.0).

This result is in the latent space very close to the embedding of "wizard" (a magical person). Thus the model can reason that Harry is also a wizard though the sentence never explicitly stated this. This is the magic of the attention mechanism. It allows the model to infer relationships and properties that are not explicitly mentioned in the input, by leveraging the context provided by other tokens in the sequence.
